Базовый объем задания

Необходимо реализовать RESTful WebAPI, используя платформу .Net 8 версии или выше. Технология хранения данных произвольная (желательно MongoDB). В рамках реализации необходимо обеспечить изоляцию/абстрагировать транспортный, логический слои и слой хранения данных (для возможности замены реализации протокола обмена данными или реализации базы данных). Нежелательно использовать библиотеку MediatR.

Необходимый функционал:
Пользователь имеет возможность управлять списками задач. Пользователь может поделиться списком задач с другим пользователем, добавив соответствующую связь. Идентификатор текущего пользователя должен передаваться вместе с запросом. Систему аутентификации и авторизации реализовывать не нужно.

Обязательные атрибуты для сущности «Список задач»:
* Название списка, длина от 1 до 255 символов.
* Пользователь, создавший список (Владелец).

Необходимые действия со списком задач:
* Создать новый список задач.
* Изменить существующий список задач.
* Удалить существующий список задач.
* Получить один существующий список задач.
* Получить список списков задач (достаточно получить ограниченный объем данных по спискам, например, только идентификатор и название списка). Должен содержать сортировку по времени создания (от новых к старым) и пагинацию.
* Добавить связь одного списка задач с указанным пользователем.
* Получить связи одного списка задач с пользователями.
* Удалить связь одного списка задач с указанным пользователем.

Правила ограничения работы с существующими списками:
* При получении списка списков задач необходимо возвращать только те сущности, где указанный в запросе пользователь либо является владельцем списка, либо имеет связь со списком задач.
* Получить список задач, Изменить список задач, Добавить связь списка задач с указанным пользователем, Получить связи списка задач с пользователями, Удалить связь списка задач с указанным пользователем может владелец списка или пользователь, имеющий связь со списком задач.
* Удалить список задач может только владелец списка.
